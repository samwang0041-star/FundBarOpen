# FundBar

把基金和股票放进 macOS 菜单栏。抬眼就能看到涨跌、估值和当日盈亏，不用反复拿手机，也不用切到行情 App。

## 这软件适合谁

如果你有下面这些习惯，FundBar 会比较合适：

- 上班时主要盯 Mac，希望少一次切屏、少一次解锁手机
- 只关心少量自选，不需要重型行情终端
- 同时持有基金和 A 股，想放在一个轻量入口里看
- 想知道今天大概赚了还是亏了，而不是只看涨跌幅

如果你需要港股、美股、分时图、交易下单或复杂分析，这个版本还不适合。

## 你能得到什么

- 菜单栏直接显示主资产涨跌幅，抬眼就能看
- 支持基金和 A 股混合管理，最多 5 个自选
- 可指定一只“主显示”，状态栏优先展示它
- 输入持仓后，可直接看到当日盈亏估算
- 基金盘中给出本地估算，官方净值发布后自动切换
- 支持 iCloud 同步，多台 Mac 登录同一账号可共享自选
- 支持开机启动和应用内更新提示

## 当前支持范围

- 基金：支持 6 位基金代码
- 股票：支持 A 股 6 位股票代码
- 股票代码输入：支持 `600519`，也兼容 `sh600519` / `sz000858`
- 系统要求：macOS 14.0 及以上

## 安装

1. 打开 [Releases](../../releases/latest) 下载最新的 DMG 安装包
2. 将 `FundBar.app` 拖到 `Applications`
3. 首次启动后，它会出现在 macOS 菜单栏

> 建议优先从 Releases 下载正式安装包；如果你是从源码自行编译，需要在 Xcode 中配置自己的签名信息。

## 3 分钟上手

1. 点击菜单栏图标，打开面板
2. 点击“添加资产”
3. 选择基金或股票
4. 输入代码并确认
5. 可选填写持仓，用于计算当日盈亏
6. 把最关心的一只设为主显示

添加完成后，菜单栏会优先展示主资产的核心信息，面板里再看完整自选列表。

## 数据说明

FundBar 追求的是“够快看一眼”，不是替代专业交易终端，所以不同资产展示的含义不一样：

当前基金和 A 股数据基于东方财富 / 天天基金的公开接口整理。

| 展示状态 | 你看到的是什么 |
| --- | --- |
| 实时 | 股票盘中实时行情 |
| 本地估算 | 基金盘中根据最近披露持仓做的参考估算 |
| 盘前估算 | 开盘前基于上一交易日信息给出的预估 |
| 官方净值 | 基金公司发布后的正式净值 |

这意味着：

- 股票盘中更适合看即时变化
- 基金盘中更适合看方向和大致盈亏
- 收盘后的基金数据会逐步切换为官方净值

## 为什么很多人会需要它

大多数行情 App 的问题不是“信息不够多”，而是“看一眼的成本太高”。

你可能只是想知道：

- 今天主力持仓涨了还是跌了
- 手里的基金大概赚了多少
- 现在要不要继续关注盘面

FundBar 的做法很直接：把最常看的那一层信息，前移到菜单栏。

## 常见问题

**Q: 支持哪些股票？**  
A: 当前支持沪深 A 股。输入 6 位代码即可，也兼容带 `sh` / `sz` 前缀的写法。

**Q: 支持哪些基金？**  
A: 当前支持 6 位基金代码的开放式基金。

**Q: 数据多久刷新一次？**  
A: 交易时段内会自动刷新，也可以手动点击“立即刷新”。

**Q: 为什么基金显示的是“本地估算”？**  
A: 因为基金盘中没有官方实时净值。FundBar 会基于最近披露持仓和市场行情做参考估算，方便你看方向和大致盈亏。

**Q: 换了 Mac 后数据还在吗？**  
A: 如果两台设备登录同一个 iCloud 账号，且系统允许同步，自选列表会自动同步。

**Q: 安全吗？**  
A: 它不会要求注册账号，没有广告、没有会员墙，也不会因为不打赏而限制功能。为了拉取行情、检查更新和可选 iCloud 同步，应用会访问对应网络服务。
